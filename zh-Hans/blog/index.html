g++ 命令详解

2025年11月21日 · 阅读需 8 分钟

g++ 是 GNU C++ 编译器的前端命令，用于编译和链接 C++ 程序。

基本语法​

g++ [选项] 源文件 [选项] [-o 输出文件]

常用编译选项​

1. 基础选项​

# 编译并链接成可执行文件
g++ main.cpp

# 指定输出文件名
g++ main.cpp -o myprogram

# 只编译不链接（生成 .o 文件）
g++ -c main.cpp

# 编译多个源文件
g++ main.cpp utils.cpp graphics.cpp -o myapp

2. 预处理选项​

# 只进行预处理，输出到标准输出
g++ -E main.cpp

# 预处理后保存到文件
g++ -E main.cpp > main.i

# 定义宏
g++ -DDEBUG main.cpp
g++ -DVERSION=100 main.cpp

3. 优化级别​

# 无优化（调试用）
g++ -O0 main.cpp

# 基本优化
g++ -O1 main.cpp

# 标准优化
g++ -O2 main.cpp

# 大量优化
g++ -O3 main.cpp

# 优化大小
g++ -Os main.cpp

# 激进优化（可能改变程序行为）
g++ -Ofast main.cpp

4. 调试信息​

# 生成调试信息
g++ -g main.cpp

# 调试信息级别
g++ -g1 main.cpp  # 最小信息
g++ -g2 main.cpp  # 默认级别
g++ -g3 main.cpp  # 最大信息（包含宏定义）

# 同时优化和调试
g++ -O2 -g main.cpp

5. 警告选项​

# 开启所有警告
g++ -Wall main.cpp

# 开启更多警告
g++ -Wall -Wextra main.cpp

# 将警告视为错误
g++ -Wall -Werror main.cpp

# 特定警告
g++ -Wunused-variable -Wshadow main.cpp

# 关闭所有警告
g++ -w main.cpp

6. 语言标准​

# C++ 标准
g++ -std=c++11 main.cpp
g++ -std=c++14 main.cpp
g++ -std=c++17 main.cpp
g++ -std=c++20 main.cpp
g++ -std=c++23 main.cpp

# GNU 扩展
g++ -std=gnu++17 main.cpp

7. 头文件和库文件​

# 指定头文件搜索路径
g++ -I/include/path main.cpp
g++ -I/usr/local/include main.cpp

# 指定库文件搜索路径
g++ -L/lib/path main.cpp
g++ -L/usr/local/lib main.cpp

# 链接库
g++ main.cpp -lmylib
g++ main.cpp -lpthread -lm

# 指定静态库
g++ main.cpp /path/to/libmylib.a

高级选项详解​

1. 架构和 CPU 优化选项​

指令集架构优化​

# 针对特定架构生成代码
g++ -march=haswell main.cpp        # Intel Haswell
g++ -march=skylake main.cpp        # Intel Skylake
g++ -march=zen2 main.cpp           # AMD Zen2
g++ -march=native main.cpp         # 自动检测当前CPU

# 针对当前机器最优编译（推荐）
g++ -march=native -O3 main.cpp

微架构调优​

# 优化指令调度
g++ -mtune=corei7 main.cpp         # 针对 Core i7 调优
g++ -mtune=generic main.cpp        # 通用调优

# 同时指定架构和调优
g++ -march=haswell -mtune=skylake main.cpp

2. 链接器控制选项​

通过 -Wl 传递选项给链接器​

# 控制堆栈大小
g++ -Wl,-stack_size,0x1000000 main.cpp    # 16MB堆栈

# 移除未使用符号
g++ -Wl,--gc-sections main.cpp

# 生成映射文件
g++ -Wl,-Map,output.map main.cpp

# 设置入口点
g++ -Wl,-e,my_entry main.cpp

# 控制符号可见性
g++ -Wl,--export-dynamic main.cpp

库搜索和链接控制​

# 指定库的搜索路径顺序
g++ -Wl,-rpath,/custom/lib main.cpp

# 延迟加载库
g++ -Wl,--as-needed main.cpp

# 静态链接特定库
g++ -Wl,-Bstatic -lstaticlib -Wl,-Bdynamic -ldynamiclib main.cpp

3. 位置无关代码 (PIC/PIE)​

共享库编译​

# 编译位置无关代码（共享库必须）
g++ -fPIC -shared -o libmylib.so mylib.cpp

# 不同PIC级别
g++ -fPIC main.cpp                 # 完全位置无关
g++ -fPIE main.cpp                 # 可执行文件位置无关
g++ -fpic main.cpp                 # 较小范围的位置无关

安全增强​

# 启用完整ASLR保护
g++ -fPIE -pie main.cpp            # 位置无关可执行文件

# 现代安全编译选项
g++ -fPIE -pie -Wl,-z,relro,-z,now main.cpp

4. 高级优化选项​

链接时优化 (LTO)​

# 启用LTO
g++ -flto -O2 main.cpp utils.cpp

# 并行LTO
g++ -flto=4 -O2 main.cpp utils.cpp

# 不同LTO级别
g++ -flto -O2 main.cpp             # 完整LTO
g++ -flto=thin -O2 main.cpp        # 瘦身LTO（内存占用少）

过程间优化​

# 单一文件的跨过程优化
g++ -fipa-pta main.cpp             # 指针分析
g++ -fipa-cp main.cpp              # 常量传播
g++ -fipa-cp-clone main.cpp        # 克隆适合的函数

# 内联控制
g++ -finline-functions main.cpp    # 内联小函数
g++ -finline-limit=200 main.cpp    # 设置内联大小限制

5. 诊断和分析选项​

代码生成报告​

# 优化报告
g++ -fopt-info main.cpp            # 所有优化信息
g++ -fopt-info-missed main.cpp     # 未成功的优化
g++ -fopt-info-vec main.cpp        # 向量化信息
g++ -fopt-info-loop main.cpp       # 循环优化信息

# 保存到文件
g++ -fopt-info-vec-optimized vec.log main.cpp

映射文件详解​

什么是映射文件？​

映射文件记录了程序内存布局的详细信息，包括：

  • 各个符号（函数、变量）在内存中的地址
  • 段（section）的大小和位置
  • 库依赖关系
  • 内存使用统计

生成映射文件的方法​

# 基本方法
g++ -Wl,-Map,output.map main.cpp -o program

# 与其它选项结合
g++ -Wl,-Map,program.map,-cref main.cpp -o program

# 在链接阶段指定
ld -Map=program.map ... object_files ... -o program

映射文件的主要内容​

内存段布局​

Section Layout:
Address  Size        Segment Section
00001000 000000c4    .text   .text
00002000 00000020    .data   .data
00003000 00000010    .bss    .bss

符号地址表​

Symbols:
Address  Size        Name
00001000 00000050    main
00001050 00000030    calculate
00002000 00000004    global_var
00002004 00000004    static_var

库和模块信息​

Archive member included because of file (symbol)
libc.a(libc-start.o)              (__libc_start_main)
libm.a(s_sin.o)                   (sin)

实际应用场景​

1. 调试复杂的内存问题​

# 编译时生成映射文件
g++ -Wl,-Map,debug.map -g main.cpp -o program

# 当程序崩溃时，通过崩溃地址在映射文件中定位问题函数

2. 分析程序大小和优化​

// 示例程序：main.cpp
#include <iostream>
#include <vector>

const int BUFFER_SIZE = 1024;
char global_buffer[BUFFER_SIZE];

void big_function() {
    std::vector<int> large_vec(1000);
    // ... 大量代码
}

int main() {
    big_function();
    return 0;
}

编译并分析：

g++ -Wl,-Map,size.map main.cpp -o program

在映射文件中可以看到：

.text          0x0000000000401000    0x1500
 .text          0x0000000000401000      0x85 main
 .text          0x0000000000401100     0x1200 big_function    # 这个函数很大！

.data           0x0000000000502000     0x1000
 .data          0x0000000000502000     0x400 global_buffer    # 全局变量大小

3. 嵌入式开发中的内存规划​

# 交叉编译时特别重要
arm-none-eabi-g++ -Wl,-Map,firmware.map,-Tlinker_script.ld main.cpp -o firmware.elf

实际应用示例​

1. 基本编译流程​

# 1. 预处理
g++ -E main.cpp -o main.i

# 2. 编译为汇编
g++ -S main.i -o main.s

# 3. 汇编为目标文件
g++ -c main.s -o main.o

# 4. 链接为可执行文件
g++ main.o -o main

# 或者一步完成
g++ main.cpp -o main

2. 多文件项目编译​

# 分别编译
g++ -c main.cpp -o main.o
g++ -c utils.cpp -o utils.o
g++ -c graphics.cpp -o graphics.o

# 链接
g++ main.o utils.o graphics.o -o myapp

# 或者一步完成
g++ main.cpp utils.cpp graphics.cpp -o myapp

3. 使用外部库（以 SFML 为例）​

# 假设 SFML 安装在标准路径
g++ -c main.cpp -o main.o
g++ main.o -o game -lsfml-graphics -lsfml-window -lsfml-system

# 如果库在非标准路径
g++ -I/path/to/sfml/include -c main.cpp -o main.o
g++ -L/path/to/sfml/lib main.o -o game -lsfml-graphics -lsfml-window -lsfml-system

4. 调试版本编译​

g++ -g -O0 -Wall -Wextra -std=c++17 main.cpp -o myapp_debug

5. 发布版本编译​

g++ -O3 -DNDEBUG -std=c++17 main.cpp -o myapp_release

高级用法组合​

高性能计算编译​

g++ -march=native -O3 -ffast-math -funroll-loops \
    -flto -DNDEBUG \
    main.cpp -o hpc_app

安全敏感应用编译​

g++ -O2 -fstack-protector-strong -D_FORTIFY_SOURCE=2 \
    -fPIE -pie -Wl,-z,relro,-z,now \
    -Wall -Wextra -Werror \
    main.cpp -o secure_app

最小体积发布​

g++ -Os -ffunction-sections -fdata-sections \
    -Wl,--gc-sections -fno-exceptions -fno-rtti \
    -DNDEBUG \
    main.cpp -o minimal_app

环境变量​

影响 g++ 的环境变量​

# C++ 包含路径
export CPLUS_INCLUDE_PATH=/custom/include:$CPLUS_INCLUDE_PATH

# 库路径
export LIBRARY_PATH=/custom/lib:$LIBRARY_PATH

# 运行时库路径
export LD_LIBRARY_PATH=/custom/lib:$LD_LIBRARY_PATH

实用技巧​

查看编译器信息​

g++ --version
g++ -v
g++ -dumpversion
g++ -dumpmachine

查看预定义宏​

g++ -dM -E - < /dev/null

依赖关系生成​

# 生成依赖关系
g++ -M main.cpp
g++ -MM main.cpp    # 忽略系统头文件

# 保存到文件
g++ -MM main.cpp > main.d

完整项目编译示例​

g++ -std=c++17 \
    -O2 \
    -Wall -Wextra \
    -I./include \
    -I/usr/local/include \
    -L./lib \
    -L/usr/local/lib \
    main.cpp \
    src/utils.cpp \
    src/graphics.cpp \
    -o myapp \
    -lsfml-graphics \
    -lsfml-window \
    -lsfml-system \
    -lpthread

这个命令：

  • 使用 C++17 标准
  • 启用 O2 优化
  • 开启所有警告
  • 添加多个头文件和库文件搜索路径
  • 编译多个源文件
  • 链接多个库
  • 指定输出文件名

掌握这些选项可以帮助您更有效地使用 g++ 进行 C++ 项目开发。

Sigmoid 激活函数 公式: $f(x) = 1 / (1 + e^{-x})$ 输出范围: (0, 1)

def sigmoid(x):
    """
    Sigmoid 激活函数
    公式: f(x) = 1 / (1 + e^(-x))
    输出范围: (0, 1)
    """
    return 1 / (1 + np.exp(-x))

def sigmoid_derivative(x):
    """Sigmoid 函数的导数"""
    s = sigmoid(x)
    return s * (1 - s)
